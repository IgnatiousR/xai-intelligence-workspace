# Fix: `react-hooks/immutability` lint error in ConnectionLines.tsx

## Problem

`linePositions` is created with `useMemo(() => new Float32Array(LN * 6), [])` and then mutated inside `useFrame`. The `react-hooks/immutability` lint rule flags this because local variables captured during render should not be reassigned/mutated after render completes.

## Fix

Change `linePositions` from `useMemo` to `useRef`. Refs are explicitly designed for mutable values that persist across renders without triggering re-renders — this is the same pattern used for `sharedPositions` in `CanvasInner.tsx:16`.

### Changes to `src/components/hero/ConnectionLines.tsx`

1. **Remove `useMemo` import** (no longer needed):
   ```diff
   - import { useRef, useMemo, type RefObject } from "react";
   + import { useRef, type RefObject } from "react";
   ```

2. **Replace `useMemo` with `useRef`** and unwrap `.current` everywhere:
   ```diff
   - const linePositions = useMemo(() => new Float32Array(LN * 6), []);
   + const linePositions = useRef<Float32Array>(new Float32Array(LN * 6));
   ```

3. **Update all `linePositions[...]` references** inside `useFrame` to use `linePositions.current[...]`:
   ```diff
   - linePositions[i * 6] = p[bA * 3];
   + linePositions.current[i * 6] = p[bA * 3];
   ```
   (Apply to all 6 assignments, lines 52-57)

4. **Update the JSX `<bufferAttribute>` to pass `linePositions.current`**:
   ```diff
   - args={[linePositions, 3]}
   + args={[linePositions.current, 3]}
   ```

### Same fix for `WowConnectionLines.tsx` (has identical pattern)

Apply the exact same changes to `src/components/wow-moment/WowConnectionLines.tsx` (lines 20, 47-52, 64).
