# **Thank you for your interest in RacoAI.**

It reflects how we think about building thoughtful, high-quality product experiences through clarity, intention, and craft. We encourage you to enjoy the process, make deliberate design decisions, and demonstrate how you bring ideas to life through clean implementation and purposeful motion.

### **Duration: 5 days**

**“The count down for the deadline starts from the day you received the Email”**

### **Goal**

Design and build a **high-fidelity interactive product experience** for Xai that demonstrates:

* UI/UX clarity  
* Design-to-code execution  
* Advanced motion & 3D interaction  
* Engineering discipline and polish

This is **not** a landing page clone.  
This is a **product-quality interactive UI** with intentional motion.

## **High-Level Overview**

You will design and implement a **single-page interactive product experience** that visually explains how Xai turns raw data into insights.

The experience should feel:

* Calm but powerful  
* Technically confident  
* Designed for decision-makers  
* Clearly “AI product” — not marketing fluff

Think: **Stripe \+ Linear \+ Vercel-level polish**, but with **custom motion & 3D depth**.

## **Product Concept (What You’re Building)**

### **Product Name (use this)**

**Xai – Intelligence Workspace**

### **Core Narrative**

“From raw data → structured intelligence → actionable insight → AI Automations”

The UI should visually *walk the user through this transformation* using animation, geometry, and motion.

## **Required Sections (Single Page)**

### **1\. Hero Section — *Data → Intelligence***

* Minimal headline \+ subtext  
* Centerpiece animated visual:  
  * Abstract geometry or particles representing “raw data”  
  * Smooth transformation into a structured form (grid, nodes, panels)  
* Motion must **respond to scroll or cursor**  
* Use:  
  * **Three.js** (preferred) OR advanced SVG \+ GSAP  
  * **Framer Motion** for UI choreography

🚫 No stock illustrations  
🚫 No Lottie files

### **2\. Interactive Insight Flow**

A horizontally or vertically animated section that explains **3 stages**:

1. Ingest Data  
2. Analyze with AI  
3. Generate Insight

Each stage should:

* Animate in/out based on scroll  
* Have micro-interactions (hover, focus, transitions)  
* Use **geometry-based animation** (lines, masks, transforms)

This is where **GSAP \+ ScrollTrigger** or **Framer Motion \+ layout animations** should shine.

### **3\. Intelligence Dashboard Preview**

A **mock product UI**, not marketing cards.

Include:

* Sidebar navigation  
* Main content panel  
* Charts / cards / tables (static data is fine)

Add:

* Subtle entrance animations  
* Hover feedback  
* State transitions (e.g. switching tabs)

Focus on:

* Visual hierarchy  
* Spacing  
* Calm, professional typography

### **4\. Signature Interaction (WOW Moment)**

One **deliberate, impressive interaction**, for example:

* A 3D object that reacts to scroll  
* A geometry morph triggered by user action  
* A data cluster that reorganizes itself  
* A depth-based parallax system

This should feel:

“This person understands motion, math, and intent.”

## **Design Requirements (Figma)**

You must create:

* A **high-fidelity Figma design**  
* Proper use of:  
  * Auto Layout  
  * Components & variants  
  * Consistent spacing system  
* Desktop-first (responsive thinking is a plus)

## **Engineering Requirements (Next.js)**

* Next.js (App Router preferred but not required)  
* Clean component structure  
* Reusable UI components  
* Thoughtful animation architecture  
* No backend needed (mock data is fine)

Animation stack:

* **Framer Motion** (required)  
* **GSAP** (for advanced timelines / scroll)  
* **Three.js or React Three Fiber** (at least one meaningful use)

🚫 No excessive pages  
🚫 No shallow animations

This is about **taste, restraint, and execution**.
